Section D

Functional Requirements

What the system must do.

ID Requirement Priority
FR-01 All pages must be navigable from the main navigation bar HIGH
FR-02 Home page must display short previews of all major sections with links to full pages HIGH
FR-03 Contact form must save submitted data to the database HIGH
FR-04 Admin panel must be protected by a login — accessible only by the owner HIGH
FR-05 Admin panel must allow full CRUD (create, read, update, delete) for all dynamic content HIGH
FR-06 Admin panel must display all form responses submitted by visitors HIGH
FR-07 Journey page must display events as a timeline with event cards in chronological order HIGH
FR-08 Journey event cards must support links to related projects, articles and external content MEDIUM
FR-09 Services page must display related project previews (images/videos) per service with live links HIGH
FR-10 Social links must be present — GitHub, LinkedIn, LeetCode and others HIGH
FR-11 All dynamic content changes made in the admin panel must reflect immediately on the frontend HIGH
FR-12 Articles must be publishable and manageable through the admin panel HIGH
FR-13 Testimonials must be addable and manageable through the admin panel MEDIUM
FR-14 Achievements must be addable and manageable through the admin panel MEDIUM
FR-15 Dark mode must be default. Light mode toggle must be available on every page HIGH
FR-16 Page transitions, scroll animations, hover effects and a page loader must be implemented HIGH
Section D

Non-Functional Requirements

How the system must behave.

ID Requirement Priority
NF-01 The design must be visually unlike any standard portfolio — cinematic, luxury, editorial feel inspired by Vogue, Christopher Nolan films and Apple HIGH
NF-02 Every visitor must feel impressed on the first landing — the Hero section must deliver a wow effect HIGH
NF-03 Desktop only for initial release. Mobile support to be considered post-launch MEDIUM
NF-04 Accent color to be decided during the UI Design stage — not locked in requirements MEDIUM
NF-05 Dark and light mode must use the same layout — only colors change, not structure HIGH
NF-06 All code must be clean, readable and structured for easy future maintenance HIGH
NF-07 Admin panel must be simple to use — no technical knowledge required to manage content HIGH
NF-08 No special performance requirements for initial release LOW
